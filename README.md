# Scan Insertion
One of the main DFT projects performed and mentioned in my CV/Resume. # 1

# Project Overview
Scan insertion to Simulation & Verification is performed on a sequential design during VLSI DFT training.

# Specifications
1. Number of flipflops : ~15K flipflops
2. Shift frequency     : 30MHZ
3. Tools used          : Cadence – Genus, Modus, Conformal, Xcelium/ Simvision
4. No.of Clocks        : Single Clock
5. Flipflops type      : Positive edge triggered
6. Time period         : 33.33ns
7. Location            : Semicon Technolabs

# Roles and Resposibilities
1. Performed Block/IP-level Scan Insertion
2. Fixed Scan DRC violations and stitched Scan chains
3. Integrated EDT compression logic
4. Performed LEC (pre vs post-scan) using Conformal
5. Generated ATPG patterns (stuck-at & at-speed)
6. Verified sample patterns and released Scan netlists.
7. Conducted coverage analysis, simulation & debug in Xcelium/SimVision

# DFT Flow
1. Read synthesized gate-level netlist and standard cell libraries.
2. Define DFT specifications including Test Mode, Test Clocks, and Scan Enable signals.
3. Perform DFT rule checks and analyze scan-register reports.
4. Execute DFT DRC checks and fix scan-related violations.
5. Insert and connect Scan Chains using Cadence Modus.
6. Generate scan-inserted netlist and validate functionality using LEC.
7. Integrate EDT (Embedded Deterministic Test) / scan compression logic.
8. Perform post-EDT LEC verification.
9. Generate ATPG patterns for Stuck-at and At-Speed faults.
10. Verify ATPG patterns and analyze fault coverage reports.
11. Release final EDT-compressed netlist for testing.
12. Perform simulation and verification using Xcelium/SimVision.

# Result
Successfully implemented complete DFT flow including Scan Insertion, EDT integration, ATPG pattern generation, and post-DFT verification using Cadence Modus flow. Achieved successful scan chain connectivity, DFT DRC closure, and validated design functionality through LEC and simulation in Xcelium/SimVision. Generated and verified Stuck-at and At-Speed ATPG patterns with satisfactory fault coverage.
